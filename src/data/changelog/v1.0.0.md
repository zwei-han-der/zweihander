## Funcionalidades Menores

- Scrollbar personalizada com estilo `thin` e bordas arredondadas para se encaixar no design do site
- Plano de fundo dinâmico que muda a cor ao recarregar a página, funciona via CSS custom properties que são alteradas via js; Temas são pré-definidos e importada de `src/data/themes.js`

  ```css
  background:
    radial-gradient(at 81% 13%, var(--color01) 0px, transparent 50%),
    radial-gradient(at 16% 91%, var(--color02) 0px, transparent 50%),
    radial-gradient(at 24% 16%, var(--color03) 0px, transparent 50%),
    radial-gradient(at 66% 4%, var(--color04) 0px, transparent 50%),
    var(--color01);
  ```

- Estilo principal sendo o glassmorphism. Inspirado pela nova versão dos aparelhos Apple com Liquid Glass
- Páginas definidas em 2 layouts principais, **Default** e **Standalone**.
- O **Default** não deveria ser chamado de default pois ele só é utilizado na Home, porém como foi o primeiro que eu fiz ficou com esse nome, conta com:
  - Grid CSS de 2 colunas, sidebar ocupa 27.5% e o restante é ocupado pelo container principal
  - Sidebar consome os componentes `MusicPlayer` e `TOC`
- O **Standalone** é o mais utilizado, é usado em todas as páginas com exceção somente da Home, conta com:
  - Mesma estrutura de Grid do Default
  - Utiliza somente o componente `TOC`
- **Modal**, que foi atualizado como componente próprio, conta com as seguintes funcionalidades:
  - Renderizado via `React.createPortal` para evitar problemas de z-index e overflow
  - Fecha via botão de fechar (o X no canto superior direito do modal), tecla Esc do teclado e clicando em qualquer lugar fora do modal
- **MarkdownRenderer** é o componente central para renderização de Markdown
  - Usa `ReactMarkdown` com os plugins `remark-gfm` e `rehype-highlight`.
  - O tema utilizado para os blocos de código é o gradient-dark importado via css
  - Todos os componentes são customizados via classes css:
    | Componente | Customização |
    | --- | --- |
    | `h1`–`h6` | Classes `.md-h1` a `.md-h6` com tamanhos, pesos e bordas inferiores distintos |
    | `p` | Detecta se o único filho é um `<img>` e renderiza como `<div>` em vez de `<p>` para evitar HTML inválido (gambiarra) |
    | `a` | Sempre abre em `target="_blank"` com `rel="noopener noreferrer"` |
    | `code` | Distingue inline vs block via prop `inline`, possuem estilos diferentes |
    | `pre` | Substituído por wrapper customizado com header (nome da linguagem + botão copiar) |
    | `input` | Sub-componente `MarkdownInput`: checkboxes GFM renderizados como `readOnly` |
    | `img` | Renderiza com `max-width: 100%` centralizado |
    | `table`, `th`, `td` | Estilos com borders, alternância de fundo em linhas estilo zebra |
